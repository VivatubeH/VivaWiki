    Coding Test 로드맵¶

    목표: 백엔드 신입 취업에 필요한 코딩테스트를 대부분 무난하게 통과할 수 있는 실력을 만든다
    언어: Java
    플랫폼: LeetCode + 프로그래머스
    학습 방식: Claude로 개념/코드 학습 -> LeetCode로 패턴 훈련 -> 프로그래머스로 국내형 실전 적용

    1. 사이트 역할 분리¶

    LeetCode¶

    • 역할: 자료구조/알고리즘 패턴 학습
    • 강점:
      • 태그 기반으로 문제를 모아 풀기 좋음
      • 공식 Study Plan이 잘 되어 있음
      • 배열, 해시, 투 포인터, 슬라이딩 윈도우, 트리, 그래프, DP 같은 패턴 훈련에 강함
    • 너한테 맞는 용도:
      • 새 개념을 배운 직후 유형 감각을 익히는 메인 사이트
      • Easy -> Medium으로 자연스럽게 난이도 올리기

    프로그래머스¶

    • 역할: 국내 기업형 코테 감각 훈련
    • 강점:
      • 공식 코딩테스트 고득점 Kit가 있음
      • 해시, 스택/큐, 힙, 정렬, 완전탐색, 탐욕법, DFS/BFS, 이분탐색, 그래프 등 코테 빈출 카테고리로 정리됨
      • SQL 문제도 같이 준비 가능
    • 너한테 맞는 용도:
      • LeetCode에서 익힌 패턴을 한국형 문제에 적용
      • 실전 문제 세트 감각 만들기
      • SQL 코테 병행

    2. 리트코드 처음 시작하는 방법¶

    처음 세팅¶

    1. 회원가입
    2. 언어를 Java로 고정
    3. Problems 탭에서 태그/난이도 필터 익히기
    4. Study Plan에서 Top Interview 150 저장
    5. 연습할 태그별 문제를 My List나 별도 노트에 관리

    리트코드에서 주로 쓸 기능¶

    • Problems
      • 태그별 문제 검색
      • 난이도 Easy / Medium 필터
    • Study Plan
      • Top Interview 150 중심으로 진행
    • Explore / Learn
      • 개념이 약한 주제 보강용
    • Discuss / Editorial
      • 막힌 문제 해설 참고용

    리트코드 운영 원칙¶

    • 처음부터 랜덤으로 풀지 않기
    • 배운 주제 -> 해당 태그 문제 3~5개 방식으로 풀기
    • Easy만 오래 붙들지 말고, 익숙해지면 바로 Medium 입문
    • 목표는 문제 수가 아니라 패턴 인식이다

    3. 프로그래머스 활용 방법¶

    주로 쓸 영역¶

    • 코딩테스트 고득점 Kit
      • 해시
      • 스택/큐
      • 힙
      • 정렬
      • 완전탐색
      • 탐욕법
      • 동적계획법
      • DFS/BFS
      • 이분탐색
      • 그래프
    • 모든 문제
      • 필요한 카테고리/레벨 추가 풀이
    • SQL 문제
      • 백엔드 취준용 SQL 코테 대비

    프로그래머스 운영 원칙¶

    • Lv1은 너무 오래 붙잡지 않기
    • Lv2를 주력으로 삼기
    • LeetCode로 배운 패턴을 프로그래머스에서 다시 적용하기
    • SQL은 주 1회 별도 학습

    4. 전체 학습 흐름¶

    한 주제당 고정 루틴¶

    1. Claude로 개념 학습
    2. Java 예제 코드 / 템플릿 이해
    3. LeetCode로 패턴 문제 3~5개 풀이
    4. 프로그래머스로 국내형 문제 1~2개 풀이
    5. 오답 정리
    6. 2~3일 뒤 재풀이

    Claude에게 물어볼 템플릿¶

    코테준비.md의 [주제명]을 공부하려고 해. Java 사용자 기준으로 1. 개념 설명 2. 코테에서 어떤 식으로 나오는지 3. 자바 예제 코드 4. 대표 풀이 템플릿 5. 헷갈리기 쉬운 포인트 6. LeetCode/프로그래머스에서 어떤 유형 문제를 풀면 좋은지 순서로 정리해줘.

    5. 시간 투자 방식¶

    추천 비율¶

    • 개념 학습: 30%
    • 문제 풀이: 50%
    • 오답 복습: 20%

    주간 권장 시간¶

    표준 플랜¶

    • 주 10~14시간
    • 평일: 1.5시간 x 4~5일
    • 주말: 2~3시간 x 2일

    하루 예시¶

    • 개념 학습 30분
    • LeetCode 1문제 30~40분
    • 풀이 리뷰 20분
    • 오답 정리 10분

    주말 예시¶

    • 프로그래머스 2문제
    • 이번 주 오답 재풀이
    • SQL 1문제 또는 부족한 유형 보강

    8. 실제 문제를 풀 때 규칙¶

    • 한 문제당 30분 이상 막히면 힌트 또는 해설 보기
    • 해설 본 문제는 다음날 무조건 재풀이
    • 오답노트는 아래 3개만 적기
      • 내가 막힌 이유
      • 핵심 아이디어 1줄
      • 다시 풀 날짜
    • Easy만 오래 하지 말고, 익숙한 유형은 Medium으로 올리기
    • 프로그래머스는 Lv1 오래 하지 말고 Lv2 중심으로 가기
