## 정렬
데이터 집합을 적합한 순서로 배치하는 것

## 시간 복잡도별 주요 정렬 알고리즘
- N^2 : 버블 정렬(Bubble Sort), 선택 정렬(Selection Sort), 삽입 정렬(Insertion Sort)
- NlogN : 퀵 정렬(Quick Sort), 병합 정렬(Merge Sort), 힙 정렬(Heap Sort)
- 단, 퀵 정렬은 최악의 경우 O(N^2)이 되는 경우도 있어서 주의.

## 자바의 정렬 알고리즘
**Arrays.sort**
- 자바의 Arrays.sort는 8가지의 자바의 기본타입에 대해 오름차순 정렬, String 배열에 대해서는 사전순 정렬을 지원한다.
- 자바의 기본형 타입의 정렬은 Dual-Pivot Quicksort를 사용한다. ( 기본 타입은 값이 같으면 순서가 바뀌어도 의미가 없음. 메모리를 적게 쓰는 퀵 정렬이 효율적이다. )
- 평균 시간 복잡도는 O(N logN)이지만, **최악의 경우 O(N^2)** 의 시간 복잡도를 갖는다.
- stable이 보장되지 않는다.

**Collections.sort**
- 자바의 Collections.sort는 객체 타입 및 직접 정의한 클래스 등 Object 타입에 대한 정렬을 지원한다.
- 자바의 객체 정렬은 Tim Sort를 사용한다.
- 평균 시간 복잡도도 O(N logN)이고, **최악의 경우 O(N logN)** 을 보장한다.
- stable하다.

## 자바의 Comparable, Comparator
- **Comparable** : **객체 스스로** 정렬 기준을 가지게 하는 인터페이스이다. 즉, **클래스의 기본 정렬 기준**을 설정할 때 사용한다. compareTo(T o) 메서드를 구현해야 한다.
- **Comparator** : **이번 경우에만 특별한 기준으로 정렬**을 할 수 있게 해주는 인터페이스이다. 즉, **기본이 아닌 다른 기준**으로 정렬하고 싶을 때 사용한다. compare(T o1, T o2) 메서드를 구현해야 한다.

## 내가 만든 클래스에 정렬 기준 구현하기
```java
class Staff {
    String name;
    int score;

    Staff(String name, int score) {
        this.name = name;
        this.score = score;
    }
}
```
1. **Comparable 인터페이스 구현** : 클래스 자체에 implements Comparable<T>를 추가하고 내부에 compareTo를 구현한다.
```java
class Staff implements Comparable<Staff> {
    // ... 필드, 생성자 동일 ...

    @Override
    public int compareTo(Staff other) {
        // 내림차순을 원하면 this와 other의 순서를 바꿈
        return this.name.compareTo(other.name); 
    }
}

// 사용 시
Collections.sort(staffList); // 자동으로 compareTo 기준 적용
```
2. **Comparator 사용** : 클래스를 수정하지 않고 정렬 시점에 즉석에서 기준을 정한다. (가장 많이 쓰이는 정렬 방식)
```java
// 1. 기본 람다 방식
Collections.sort(staffList, (s1, s2) -> s2.score - s1.score);

// 2. 참고) Comparator의 메서드 활용 : 점수를 비교해서, 역순으로 정렬하기.
staffList.sort(Comparator.comparingInt((Staff s) -> s.score).reversed());
```
