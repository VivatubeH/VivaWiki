## Array vs List

### Array (배열)
- **정적 구조**: 생성 시 크기 고정
- **연속 메모리**: 인덱스 접근 O(1)
- **크기 변경 불가**: 새 배열 생성 필요

### List (리스트)
- **동적 구조**: 크기 자동 조절
- **추가/삭제 자유**: 필요에 따라 확장/축소
- **인터페이스**: ArrayList, LinkedList 등 구현체 선택

## ArrayList vs LinkedList 선택 기준

### ArrayList
```java
List<Integer> list = new ArrayList<>();
```
- **내부 구조**: 동적 배열
- **장점**: 인덱스 접근 O(1)
- **단점**: 중간 삽입/삭제 O(n)
- **사용**: 조회가 많을 때

### LinkedList
```java
List<Integer> list = new LinkedList<>();
```
- **내부 구조**: 노드 연결
- **장점**: 앞/뒤 삽입/삭제 O(1)
- **단점**: 인덱스 접근 O(n)
- **사용**: 삽입/삭제가 많을 때

## 핵심 메서드 

### 추가
```java
list.add(element);           // 끝에 추가 O(1)
list.add(index, element);    // 중간 삽입 O(n)
```

### 삭제
```java
list.remove(index);          // 인덱스로 삭제 O(n)
list.remove(object);         // 객체로 삭제 O(n)
list.clear();                // 전체 삭제 O(n)
```

### 조회
```java
list.get(index);             // 인덱스 접근 O(1) - ArrayList
list.size();                 // 크기 O(1)
list.isEmpty();              // 비었는지 O(1)
```

### 검색
```java
list.contains(element);      // 포함 여부 O(n)
list.indexOf(element);       // 첫 위치 O(n)
```

---

## 시간 복잡도 비교

| 연산 | ArrayList | LinkedList |
|------|-----------|------------|
| get(i) | O(1) | O(n) |
| add(끝) | O(1) | O(1) |
| add(i, e) | O(n) | O(n) |
| remove(i) | O(n) | O(n) |
| remove(처음) | O(n) | O(1) |

## 알고리즘 문제 패턴

### 1. 순회
```java
// for-each (가장 빠름)
for (int x : list) {
    System.out.println(x);
}

// 인덱스 사용
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

### 2. 중간 삭제 (뒤에서부터)
```java
// ❌ 앞에서부터 삭제 (인덱스 밀림)
for (int i = 0; i < list.size(); i++) {
    if (조건) list.remove(i);  // 버그 발생
}

// ✅ 뒤에서부터 삭제
for (int i = list.size() - 1; i >= 0; i--) {
    if (조건) list.remove(i);
}
```

### 3. LinkedList 양방향 활용
```java
LinkedList<Integer> deque = new LinkedList<>();
deque.addFirst(1);   // 앞에 추가 O(1)
deque.addLast(2);    // 뒤에 추가 O(1)
deque.removeFirst(); // 앞에서 삭제 O(1)
deque.removeLast();  // 뒤에서 삭제 O(1)
```

## 핵심 정리

### 선택 기준
- **조회 많음** → ArrayList
- **삽입/삭제 많음** → LinkedList
- **양쪽 추가/삭제** → LinkedList (Deque처럼 사용)

### 주의사항
- **중간 삭제**: 뒤에서부터 또는 Iterator
- **크기 변경**: remove() 후 size() 재확인
- **원형 처리**: `idx % list.size()`

**실전에서는 ArrayList 90%, LinkedList 10%**
