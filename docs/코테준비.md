# Coding Test 로드맵
> 목표: 백엔드 신입 취업에 필요한 코딩테스트를 대부분 무난하게 통과할 수 있는 실력을 만든다  
> 언어: Java  
> 플랫폼: LeetCode + 프로그래머스  
> 학습 방식: `Claude로 개념/코드 학습 -> LeetCode로 패턴 훈련 -> 프로그래머스로 국내형 실전 적용`

---

## 1. 사이트 역할 분리

### LeetCode
- 역할: 자료구조/알고리즘 `패턴 학습`
- 강점:
    - 태그 기반으로 문제를 모아 풀기 좋음
    - 공식 `Study Plan`이 잘 되어 있음
    - 배열, 해시, 투 포인터, 슬라이딩 윈도우, 트리, 그래프, DP 같은 패턴 훈련에 강함
- 너한테 맞는 용도:
    - 새 개념을 배운 직후 `유형 감각`을 익히는 메인 사이트
    - Easy -> Medium으로 자연스럽게 난이도 올리기

### 프로그래머스
- 역할: `국내 기업형 코테 감각` 훈련
- 강점:
    - 공식 `코딩테스트 고득점 Kit`가 있음
    - 해시, 스택/큐, 힙, 정렬, 완전탐색, 탐욕법, DFS/BFS, 이분탐색, 그래프 등 코테 빈출 카테고리로 정리됨
    - SQL 문제도 같이 준비 가능
- 너한테 맞는 용도:
    - LeetCode에서 익힌 패턴을 한국형 문제에 적용
    - 실전 문제 세트 감각 만들기
    - SQL 코테 병행

---

## 2. 리트코드 처음 시작하는 방법

### 처음 세팅
1. 회원가입
2. 언어를 `Java`로 고정
3. `Problems` 탭에서 태그/난이도 필터 익히기
4. `Study Plan`에서 `Top Interview 150` 저장
5. 연습할 태그별 문제를 `My List`나 별도 노트에 관리

### 리트코드에서 주로 쓸 기능
- `Problems`
    - 태그별 문제 검색
    - 난이도 Easy / Medium 필터
- `Study Plan`
    - `Top Interview 150` 중심으로 진행
- `Explore / Learn`
    - 개념이 약한 주제 보강용
- `Discuss / Editorial`
    - 막힌 문제 해설 참고용

### 리트코드 운영 원칙
- 처음부터 랜덤으로 풀지 않기
- `배운 주제 -> 해당 태그 문제 3~5개` 방식으로 풀기
- Easy만 오래 붙들지 말고, 익숙해지면 바로 Medium 입문
- 목표는 문제 수가 아니라 `패턴 인식`이다

---

## 3. 프로그래머스 활용 방법

### 주로 쓸 영역
- `코딩테스트 고득점 Kit`
    - 해시
    - 스택/큐
    - 힙
    - 정렬
    - 완전탐색
    - 탐욕법
    - 동적계획법
    - DFS/BFS
    - 이분탐색
    - 그래프
- `모든 문제`
    - 필요한 카테고리/레벨 추가 풀이
- `SQL 문제`
    - 백엔드 취준용 SQL 코테 대비

### 프로그래머스 운영 원칙
- Lv1은 너무 오래 붙잡지 않기
- Lv2를 주력으로 삼기
- LeetCode로 배운 패턴을 프로그래머스에서 다시 적용하기
- SQL은 주 1회 별도 학습

## 4. 전체 학습 흐름

### 한 주제당 고정 루틴
1. `Claude`로 개념 학습
2. Java 예제 코드 / 템플릿 이해
3. LeetCode로 패턴 문제 3~5개 풀이
4. 프로그래머스로 국내형 문제 1~2개 풀이
5. 오답 정리
6. 2~3일 뒤 재풀이

### Claude에게 물어볼 템플릿
`코테준비.md의 [주제명]을 공부하려고 해.`
`Java 사용자 기준으로`
`1. 개념 설명`
`2. 코테에서 어떤 식으로 나오는지`
`3. 자바 예제 코드`
`4. 대표 풀이 템플릿`
`5. 헷갈리기 쉬운 포인트`
`6. LeetCode/프로그래머스에서 어떤 유형 문제를 풀면 좋은지`
`순서로 정리해줘.`

## 5. 시간 투자 방식

## 추천 비율
- 개념 학습: `30%`
- 문제 풀이: `50%`
- 오답 복습: `20%`

## 주간 권장 시간
### 표준 플랜
- 주 10~14시간
- 평일: 1.5시간 x 4~5일
- 주말: 2~3시간 x 2일

### 하루 예시
- 개념 학습 30분
- LeetCode 1문제 30~40분
- 풀이 리뷰 20분
- 오답 정리 10분

### 주말 예시
- 프로그래머스 2문제
- 이번 주 오답 재풀이
- SQL 1문제 또는 부족한 유형 보강

## 6. 네 커리큘럼 기반 주제별 풀이 방식

## 4-1. 자료구조

### Array
- Claude:
    - 배열 특징
    - 인덱스 접근
    - Java 배열 사용법
- LeetCode:
    - Array / String Easy 위주 3~5문제
- 프로그래머스:
    - 구현 / 배열 다루기 문제 1~2문제

### String
- Claude:
    - 문자열 처리
    - 부분문자열
    - StringBuilder
- LeetCode:
    - String, Two Pointers 입문 문제
- 프로그래머스:
    - 문자열 처리 문제 1~2문제

### LinkedList 개념
- Claude:
    - 배열과 차이
    - 연결 구조
    - 삽입/삭제 특징
- LeetCode:
    - Linked List Easy 2~3문제만
- 프로그래머스:
    - 굳이 강하게 비중 둘 필요 없음

### Hash 활용 (HashMap / HashSet)
- Claude:
    - 빈도수 세기
    - 중복 제거
    - 빠른 조회
- LeetCode:
    - Hash Table 태그 4~5문제
- 프로그래머스:
    - 고득점 Kit `해시` 1~2문제

### Stack & Queue
- Claude:
    - LIFO/FIFO
    - Java `ArrayDeque` 사용법
- LeetCode:
    - Stack / Queue 문제 3~4문제
- 프로그래머스:
    - 고득점 Kit `스택/큐` 1~2문제

### Heap (우선순위 큐)
- Claude:
    - 최소힙/최대힙
    - Java `PriorityQueue`
- LeetCode:
    - Heap 태그 2~3문제
- 프로그래머스:
    - 고득점 Kit `힙` 1문제

### Tree
- Claude:
    - 이진 트리
    - BST
    - 전위/중위/후위/레벨 순회
- LeetCode:
    - Tree Easy/Medium 3~4문제
- 프로그래머스:
    - 직접 비중은 낮지만 DFS/BFS 문제와 연결

### Graph
- Claude:
    - 인접 리스트
    - 인접 행렬
    - DFS/BFS 연결
- LeetCode:
    - Graph 기초 2~3문제
- 프로그래머스:
    - 고득점 Kit `그래프`는 후반부에 풀이

### HashMap 내부 동작
- Claude:
    - 해시 충돌
    - equals/hashCode
    - 내부 구조
- LeetCode / 프로그래머스:
    - 문제 풀이보다는 면접 보강용
- 메모:
    - 코테용보다 면접용 비중이 큼
