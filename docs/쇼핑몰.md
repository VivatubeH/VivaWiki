## 기술 스택
- Spring Boot 3.x
- Spring Security
- JWT
- JPA
- QueryDSL
- MySQL
- Redis
- 포트원(아임포트)
- Swagger

## 프로젝트 기반 설정
- 게시판과 동일한 공통 기반 구성
- Redis 연결 설정 (Refresh Token 저장용)
- 포트원 API 키 설정
- 멀티 모듈 or 레이어드 아키텍처 정리

## 회원 기능
- 회원가입 / 로그인 / 로그아웃 (JWT + Redis Refresh Token)
- 내 정보 조회 / 수정
- 비밀번호 변경
- 배송지 CRUD (기본 배송지 설정 포함)
- 회원 탈퇴

## 상품 기능
- 상품 등록 / 수정 / 삭제 (관리자 전용)
- 상품 이미지 업로드
- 상품 목록 조회 (페이지네이션, 가격순/최신순/판매량순 정렬)
- 상품 상세 조회
- 카테고리별 상품 조회
- 상품 검색 (상품명 / 가격 범위 필터)
- 재고 수량 관리
  
## 장바구니 기능
- 장바구니에 상품 추가
- 장바구니 목록 조회 (총 금액 계산 포함)
- 수량 변경
- 특정 상품 삭제
- 장바구니 전체 비우기

## 주문 기능
- 장바구니 → 주문 생성
- 단건 상품 바로 주문
- 주문 목록 조회 (내 주문 내역)
- 주문 상세 조회
- 주문 취소 (결제 전 PENDING 상태에서만)
- 반품 요청 (결제 완료 후)
- 주문 시 재고 차감 + Optimistic Lock으로 동시성 처리 ← 핵심
- 주문 완료 시 장바구니 자동 비우기

## 결제 기능
- 포트원 결제 요청 (카카오페이 테스트 모드)
- 결제 완료 후 서버사이드 금액 검증 (위변조 방지)
- 결제 성공 → 주문 상태 PAID 변경
- 결제 취소 / 환불 처리
- 결제 내역 조회

## 리뷰 기능
- 리뷰 작성 (구매 완료한 상품만, 1회 제한)
- 리뷰 수정 / 삭제
- 상품별 리뷰 목록 조회
- 상품 평균 평점 집계

## 찜 기능
- 상품 찜 추가 / 해제 (토글)
- 내 찜 목록 조회

## 관리자 기능
- 상품 등록 / 수정 / 삭제
- 주문 상태 변경 (PAID → SHIPPING → DELIVERED)
- 재고 수동 조정
- 회원 목록 조회 / 강제 탈퇴

## 기술적 구현 포인트 (면접 어필용)
- Optimistic Lock (@Version)으로 재고 동시성 문제 해결 — 가장 중요한 어필 포인트
- @Transactional 전파 속성 (REQUIRED, REQUIRES_NEW) 이해하고 활용
- QueryDSL로 상품 검색 다중 조건 동적 쿼리
- Redis에 Refresh Token 저장 및 만료 처리
- 포트원 결제 금액 서버 검증 로직
- ApplicationEventPublisher로 주문 완료 이벤트 처리 (재고 차감 분리)
- 서비스 단위 테스트 + 주문/결제 통합 테스트
- GitHub Actions로 CI 파이프라인 구성 (선택)
