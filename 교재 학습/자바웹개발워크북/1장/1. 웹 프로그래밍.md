웹 프로그래밍의 특징
-------
- 기존 프로그래밍 방식은 개발자가 모든 처리 과정을 직접 설계하는 방식이었다면,
- 웹 개발은 개발자의 코드를 정해진 구조에 맞게 넣는 방식임 ( 그에 따라 구조와 흐름에 대한 이해가 핋수적 )

웹 프로젝트의 기본구조
------------
<img width="822" height="194" alt="image" src="https://github.com/user-attachments/assets/9036362a-f5e6-46fb-a879-f03bd448aed7" />

- 브라우저 혹은 클라이언트 프로그램
  1. 서버에서 전송한 결과를 화면에 보여줌 ( 렌더링(rendering))
  2. 사용자의 화면 조작을 이용해서 서버에 원하는 데이터를 보내고 (요청-request) 받는 (응답-response) 프로그램
 
- 웹 서버(Web Server) 혹은 WAS(Web Application Server)
  1. 이미지처럼 고정된 데이터를 제공하는 서버 (웹 서버)
  2. 동적으로 매번 새로운 데이터를 만들어 낼 수 있는 서버 (WAS)
- 실제 운영환경에서는 웹 서버와 WAS를 분리해서 운영하지만, 대부분의 WAS는 웹 서버 기능도 겸하고 있음.

- 데이터베이스 : 영구적으로 데이터를 보관하고 운영하기 위한 프로그램으로, 기본적으로 관계형(relational) 패러다임을 이용한 관계형 데이터베이스를 사용함.
