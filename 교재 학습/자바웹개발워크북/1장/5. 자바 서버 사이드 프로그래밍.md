서버 사이드 프로그래밍
--------------------------------
- 서버쪽에서 프로그래밍을 통해 데이터를 처리할 수 있도록 구성하는 것.

#### 서버 사이드 프로그래밍의 고려사항
1. 동시에 여러 요청(Request) 시, 어떻게 처리할까?
2. 서버에서 문제 발생 시 어떻게 처리할까?
3. 어떤 방법으로 데이터 전송을 최적화할까?
4. 분산 환경이나 분산 처리 시 발생할 수 있는 문제점은 무엇이 있나?

- 그러나, 서비스 개발 시마다 이런 고민을 하면 개발 비용과 시간 소모가 커짐. ( 비효율 )
- 자바는 이러한 처리를 JavaEE라는 기술 스택으로 정리해두었음.

서블릿(Servlet) 기술
--------------------------
- 서버에서 동적으로 요청과 응답을 처리할 수 있는 API들을 정의한 것.
- 서블릿 지원 환경에서 개발자들은 서블릿에서 제공하는 API를 이용해서 코드를 작성, 설정하는 방식으로 서블릿 프로그램을 작성함.
- 서블릿은 '코드를 이용한 처리', JSP는 '화면 개발'로 역할을 분담해서 개발하는 것이 일반적

서블릿 컨테이너(Servlet Container)
-------------------------------
- 서블릿(Servlet)은 서블릿을 실행할 수 있는 환경에서 실행이 가능함. 이를 서블릿 컨테이너라고 함.
- 우리가 설치한 톰캣(Tomcat)이 서블릿 컨테이너(서블릿 엔진)(Servlet Container)의 역할을 수행함.

서블릿 컨테이너(Servlet Container)로 인한 자바와의 차이점
----------------------------------------------
- 객체 생성, 호출은 사용자가 아닌 서블릿 컨테이너가 함.
- 서블릿 클래스에서 생성한 객체는 서블릿 컨테이너가 관리함.
- 서블릿/JSP 코드 개발 시 기본 자바 API 뿐만 아니라, 서블릿 API도 필요함.

JSP 기술
---------------------------
- Java Server Pages의 약자
- 서블릿 기술과 동일하게 서버에서 동적으로 데이터를 구성하는 기술
- JSP 기술은 서블릿과 달리 HTML 코드를 그대로 사용하고, 필요시에만 자바 코드를 약간 넣는 방식. ( HTML을 쉽게 쓸 수 있음 )
- 서블릿 코드는 자바 코드를 이용해서 HTML 문자열을 만들어내는 방식

JSP가 서블릿과 동일하게 처리되는 이유
---------------------
- JSP 코드는 자바 코드는 아니지만, 서블릿 코드로 변환되어서 컴파일되고 실행되게 된다.
- 필요시에 자바 파일로 생성되고, 컴파일되어서 class 파일로 만들어지게 된다.
