## 1. 서버 사이드 프로그래밍 (Server Side Programming)
* **개념**: 서버 측에서 데이터를 처리하고 동적인 결과를 생성하는 프로그래밍 방식
* **Java의 해결책 (Java EE)**: 동시성 처리, 장애 대응, 최적화 등 복잡한 서버 로직을 개발자가 매번 밑바닥부터 짜지 않도록 표준화된 기술 스택(Servlet, JSP 등)을 제공

## 2. 서블릿(Servlet)과 서블릿 컨테이너
* **서블릿(Servlet)**: 서버에서 동적 요청/응답을 처리하기 위해 정의된 Java API
* **서블릿 컨테이너(Servlet Container)**: 서블릿을 실행하고 생명주기를 관리하는 엔진 (예: **Tomcat**)

### 💡 일반 자바 프로그래밍과의 결정적 차이 (중요 ⭐)
* **제어권의 주체**: 개발자가 직접 `new`로 객체를 생성하거나 메소드를 호출하지 않음
* **제어의 역전(IoC)**: **서블릿 컨테이너**가 서블릿 객체의 생성, 실행(`init`, `doGet`, `destroy`), 소멸을 모두 관리함
* **환경**: 실행을 위해 기본 자바 API 외에 추가적인 **서블릿 API** 라이브러리가 반드시 필요함

## 3. 서블릿(Servlet) vs JSP
| 구분 | 서블릿 (Servlet) | JSP (Java Server Pages) |
| :--- | :--- | :--- |
| **주 역할** | 비즈니스 로직 처리 (Controller) | 화면 구성 및 출력 (View) |
| **특징** | Java 코드 안에 HTML 문자열을 넣음 | HTML 코드 안에 Java 코드를 삽입함 |
| **본질** | 자바 클래스 파일 | **결국 서블릿 코드로 변환되어 실행됨** |

### 🔍 JSP가 실행되는 과정
1. 사용자가 `.jsp` 파일을 요청
2. 서블릿 컨테이너가 JSP 파일을 **`.java` 소스 파일로 변환**
3. 변환된 파일을 컴파일하여 **`.class` 파일(서블릿) 생성**
4. 최종적으로 서블릿으로서 실행되어 결과를 응답
