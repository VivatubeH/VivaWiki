Request(요청) / Response(응답)
----------------------------------
- 일반적으로 브라우저는 자신이 원하는 정보를 전달하기 위해서 두 가지 방식을 이용함.
- GET 방식 : 주소창에 직접 원하는 데이터를 적거나 링크를 클릭해서 호출
  1. 주소와 필요한 데이터를 한 번에 같이 보냄.
  2. 단순 링크로 처리되므로 메신저나 SNS를 통한 공유가 쉽다.
  3. 주로 특정 정보를 조회할 때 사용됨.
- POST 방식 : 입력 화면에서 필요한 내용을 작성한 후 '전송' 같은 버튼을 클릭해서 호출.
  1. 주소와 필요한 데이터를 따로 보내는 방식
  2. 주로 회원가입이나 로그인 등의 처리에 해당됨.
  3. 웹 화면을 통해 실제 처리가 필요한 작업을 하기 위해 사용됨.

- 요청(Request) : 브라우저가 서버에 앞선 방식으로 데이터를 요청한은 것.
- 응답(Response) : 서버가 Response(응답) 데이터를 만들어 브라우저로 보낸다.

서버에서의 데이터 처리 방식
-----------------------
- 정적 데이터(static data) : 항상 동일하게 고정된 데이터를 전송하는 방식, 주로 파일로 고정된 HTML, CSS, 이미지 파일 등의 데이터
- 동적 데이터(dynamic data) : 매번 필요할 때마다 다른 데이터를 동적으로 구성해서 전송하는 방식, 메일함처럼 상황에 따라 동적으로 서버에서 데이터를 만들어 보내는 방식이라 서버 사이드 프로그래밍(Server Side Programming)이라고 함.

웹 서버와 웹 어플리케이션 서버
-----------------------
- 웹 서버(Web Server) : 항상 정적(static) 데이터를 보내는 역할만을 수행하는 서버
- 웹 애플리케이션 서버(Web Application Server) : 동적 데이터(dynamic data)를 만들어 보내는 서버

- 앞에서 사용한 톰캣은 엄밀히 보면 WAS이지만, 대부분의 WAS는 웹 서버 기능도 포함하기 때문에 이미지, CSS/JS 처럼 정적 파일과 서블릿/JSP 같은 동적 자원을 모두 처리할 수 있다.

HTTP
------------------------
- 프로토콜 : 데이터 교환의 약속이자 방식
- HTTP는 웹에서 데이터를 주고 받기 위한 프로토콜이다. (Hyper Text Transfer Protocol)
- 웹을 사용하면서 http://, https://로 시작하는 주소의 의미가 HTTP, HTTPS 프로토콜을 이용해서 데이터를 주고받음을 의미함.

- HTTP 메시지는 기본적으로 헤더(Header), 몸체(Body)로 구성됨.
- 비유하면 HTTP의 헤더가 편지봉투, HTTP의 바디가 편지의 내용물 같다고 이해하면 편함.

HTTP의 비연결성(Connectionless)
----------------------------
- 웹은 특성상 여러 사용자가 브라우저를 통해 서버를 호출하는 구조임.
- 서버에서는 최대한 많은 사용자에게 서비스를 제공해야만 함.
- 그렇게 하기 위해서, HTTP는 비연결성(Connectionless) 방식을 사용함.

- 비연결성(Connectionless) : 하나의 요청과 응답을 처리한 뒤에 연결이 종료된다. ( 연결이 유지되지 않음 )
- 그렇기 때문에, 특정 웹 페이지의 최신 결과를 보려면 '새로 고침'을 해야한다. ( 이는, 브라우저와 서버의 연결이 끝나서 과거의 결과만 남아있기 때문 )

- 비연결성을 통해 하나의 요청(Request)을 빨리 처리하고 연결을 종료해서 다음 요청(Request)를 받을 수 있게 되고, 결과적으로 적은 리소스를 이용해서 많은 수의 요청(Request)들을 처리할 수 있다.
