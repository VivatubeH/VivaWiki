## 1. Request(요청) & Response(응답)
* **요청(Request)**: 브라우저가 서버에 데이터를 요구하는 행위
* **응답(Response)**: 서버가 요청에 맞는 데이터를 만들어 브라우저로 전송하는 것

### 📥 데이터 전달 방식 (GET vs POST)
| 구분 | GET 방식 | POST 방식 |
| :--- | :--- | :--- |
| **전달 방식** | 주소창에 데이터 노출 (URL+데이터) | 주소와 데이터 분리 전송 (Body에 포함) |
| **특징** | 공유(SNS, 링크)가 쉬움 | 데이터 숨김 처리가 가능함 |
| **주 용도** | 특정 정보 **조회** | **회원가입, 로그인** 등 실제 처리 작업 |

## 2. 웹 서버(WS)와 웹 애플리케이션 서버(WAS)
* **정적 데이터(Static)**: HTML, CSS, 이미지 등 고정된 파일 -> **웹 서버** 담당
* **동적 데이터(Dynamic)**: 사용자 상황에 따라 실시간 생성되는 데이터 -> **WAS** 담당
* **톰캣(Tomcat)**: 엄밀히는 WAS이지만, 웹 서버 기능도 포함하여 정적/동적 자원을 모두 처리 가능

## 3. HTTP 프로토콜의 특징
* **HTTP**: 웹에서 데이터를 주고받기 위한 약속(Hyper Text Transfer Protocol)
* **메시지 구조**: **Header(편지봉투)** - 메타데이터 / **Body(내용물)** - 실제 데이터

### ⚡ 비연결성 (Connectionless) - 중요 ⭐
* **개념**: 서버가 응답을 보낸 직후 클라이언트와의 연결을 즉시 끊는 방식
* **이유**: 한정된 서버 리소스로 최대한 많은 사용자의 요청을 처리하기 위함
* **결과**:
    1. 연결 유지 비용을 줄여 많은 수의 요청 처리 가능
