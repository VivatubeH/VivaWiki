변경된 코드 반영하기
------------------
- 예제 프로젝트는 gradle을 이용해서 컴파일 등의 작업이 처리되고, 이후 build/libs 폴더 안의 내용이 톰캣을 통해 실행되게 된다.
- 따라서, 코드를 변경한 후에는 반드시 톰캣을 재시작해야만 한다.

 <img width="650" height="379" alt="image" src="https://github.com/user-attachments/assets/f732689e-f4bc-480f-8e07-4f7275383d38" />

 - 재시작을 최소화 하기 위해 'Edit Configurations'의 'On update action'과 'On frame deactivation'을 사진처럼 조정한다.
 - 톰캣을 위한 한글 설정은 VM options를 이용해 사진처럼 진행한다.

서블릿 코드 작성하기
-------------------------
- 톰캣에서 작성하는 자바 코드는 HttpServlet이라는 클래스를 상속해서 작성한다.
- 이를, 서블릿(Servlet) 클래스를 생성한다고 표현한다.

<img width="1378" height="822" alt="image" src="https://github.com/user-attachments/assets/d3250238-fd16-4d0b-a54c-4e6d401cc220" />

- MyServlet은 HttpServlet 클래스를 상속하고, @WebServlet이라는 어노테이션을 이용해서 서블릿의 경로를 지정한다.
- @WebServlet : 브라우저의 경로와 해당 서블릿을 연결하는 설정을 위해 사용한다.
- 어노테이션(Annotation) : 코드상에 추가적인 정보를 남겨두기 위해 주로 사용한다. ( 코드에 대한 추가적인 처리나 설정을 위해 쓰임 )
- doGet() : 브라우저의 주소를 직접 변경해서 접근하는 경우에 호출되는 메소드이다.
- PrintWriter : 자바 코드에서 System.out.println()처럼 서블릿은 PrintWriter 객체를 이용해서 브라우저쪽으로 출력을 처리한다. ( 브라우저로의 출력을 위한 용도 )
