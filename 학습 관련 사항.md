- 버전 관리는 깃(Git)을 베이스로 한 깃랩이나 깃허브 엔터프라이즈 관련 제품을 사용함.
- 따라서, 깃을 이용한 개인프로젝트를 진행하는 것이 좋음.

- 테스트 기반 개발을 중요시하는 회사가 많음.
- TDD(Test Driven Development)를 학습하고 개발과 테스트 케이스 작성 연습을 하는 것이 좋음.

- DB는 MySQL을 기본으로 하고, Document 기반의 몽고DB 같은 NoSQL을 원하는 곳도 존재함.
- 주니어는 언어는 1~2가지면 충분하고 필요할 때 배우면 됨.
- 대신 ERD나 아키텍처 관련 지식은 언어와 무관하게 쓰이니 배워두는 게 좋음.
- 서버 개발을 지향하면 ANSI-SQL은 평균 이상으로 습득해야 함. ( 특정 벤더에 종속되지 않은 표준 기술 )
- 서버 애플리케이션 레벨에서 가장 많이 도입되는 기술인 ORM(Object-Relational Mapping) 기술은 반드시 배워야 함.
- 쿼리를 직접 호출하는 것보다는 ORM이나 DSL(Domain-Specific Lanaguage) 기반 개발 방식이 생산성에 직결됨.

- 포트폴리오용 토이 프로젝트보다는 실제 서비스를 론칭해서 실사용자를 대상으로 서비스를 운영해 본 경험 어필하기.
- 장기적으로 AI 도구들로 백엔드, 프런트엔드 경계가 허물어지는 추세 ( 분야에 제한을 두지 말자. )

- 클론 코딩, 토이 프로젝트는 면접관에게 별로 어필이 안 됨.
- 가능한 외부에서 접근할 수 있도록 도메인을 연결하거나 실제 앱 마켓에 등록해서 1에서 100까지 기획부터 출시, 운영까지
  경험치가 쌓일 수 있게 몰입해서 완성하기.

#### 프로젝트 포트폴리오에 포함되어야 할 사항
- 개발 목표와 의도 : 무엇을 해결하려고 했고, 무엇을 제공하려고 했는지
- 주요 기능, 시스템 개요 (아키텍처)
- 본인이 담당한 영역과 기여도
- 소스 레포지토리(repository) 주소

- 형식은 도식화(최대한 간결하게, 한 눈에 들어오게)
- 노션보다는 PDF나 구글 Docs로 두장 이내 압축
- 서비스는 외부에서 접근 가능한 상태로 처리, 앱이나 웹으로 올려본 경험이 중요

- 선택과 집중 필요, 양보단 질 ( 구색 맞추기 X )
- 나를 알리는 활동이니 나의 강점, 내가 만든 제품이나 서비스를 한 눈에 잘 알아볼 수 있게 해야한다.
