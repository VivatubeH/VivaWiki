# 상속 (Inheritance)

## 핵심 개념

**상속이란?**
- `extends` 키워드로 부모 클래스의 필드와 메서드를 자식이 물려받는 기능
- 코드 중복 제거 + 기능 확장이 목적
```java
public class Parent { }
public class Child extends Parent { }  // Child가 Parent 상속
```

**단일 상속만 가능**
- 자바는 `extends` 대상을 하나만 선택 가능
- 다이아몬드 문제 방지

---

## 메모리 구조

**객체 생성 시**
- 상속 관계의 객체를 생성하면 메모리에 부모 + 자식 모두 생성됨
- 참조값은 하나지만 내부엔 두 클래스 정보가 공존

**메서드 호출 순서**
1. 호출한 변수 타입에서 메서드 찾기 시작
2. 현재 타입에 없으면 → 부모 타입으로 올라가서 찾음
3. 최상위 부모까지 올라가도 없으면 컴파일 오류

---

## 메서드 오버라이딩

**정의**
- 부모의 메서드를 자식에서 재정의
```java
public class Parent {
    public void move() {
        System.out.println("이동");
    }
}

public class Child extends Parent {
    @Override  // 필수는 아니지만 실수 방지를 위해 권장
    public void move() {
        System.out.println("빠르게 이동");
    }
}
```

**오버라이딩 조건**
- 메서드 이름, 매개변수, 반환 타입 동일
- 접근 제어자는 부모보다 넓거나 같아야 함
- `static`, `final`, `private` 메서드는 오버라이딩 불가

---

## super 키워드

**super의 역할**
- 부모 클래스의 필드/메서드 참조
- 자식과 부모에 같은 이름이 있을 때 부모 것 호출
```java
public class Child extends Parent {
    public void call() {
        super.value;      // 부모 필드 접근
        super.method();   // 부모 메서드 호출
    }
}
```

---

## super() - 생성자

**핵심 규칙**
- 자식 생성자의 **첫 줄**에 반드시 `super(...)`로 부모 생성자 호출
- 부모의 기본 생성자 호출은 생략 가능 (자바가 자동 추가)
```java
public class Parent {
    public Parent(String name) { }
}

public class Child extends Parent {
    public Child() {
        super("이름");  // 부모에 기본 생성자 없으면 필수
    }
}
```

**생성자 실행 순서**
- 최상위 부모부터 실행 → 자식으로 내려옴
- 초기화는 부모 먼저, 자식은 나중에

---

## 접근 제어자와 상속

**상속 관계에서의 접근 범위**
- `private`: 자식도 접근 불가
- `default`: 같은 패키지 + 상속 관계 OK
- `protected`: 다른 패키지 + 상속 관계 OK ⭐
- `public`: 모두 OK

**핵심**
- 상속 관계라도 자식은 "외부"로 취급됨
- 접근 제어자의 규칙을 그대로 따름

---

## final 키워드

**클래스에 final**
```java
public final class MyClass { }  // 상속 불가
```

**메서드에 final**
```java
public final void myMethod() { }  // 오버라이딩 불가
```

## 면접 대비 핵심

**Q1. 상속이란?**
- 부모 클래스의 기능을 자식이 물려받아 재사용하는 기능. 코드 중복 제거와 확장성이 목적.

**Q2. 자바는 왜 단일 상속만 지원하나?**
- 다중 상속 시 다이아몬드 문제(같은 메서드를 가진 부모 2개) 발생. 어떤 부모 메서드를 호출할지 모호해짐.

**Q3. 오버라이딩과 오버로딩 차이?**
- 오버라이딩: 상속 관계에서 부모 메서드를 자식이 재정의
- 오버로딩: 같은 이름의 메서드를 매개변수만 다르게 여러 개 정의

**Q4. super()는 왜 생성자 첫 줄에 와야 하나?**
- 자식 객체를 초기화하기 전에 부모 객체가 먼저 초기화되어야 하므로.

**Q5. protected를 언제 쓰나?**
- 외부엔 숨기되, 상속받은 자식 클래스에서는 접근이 필요할 때.
