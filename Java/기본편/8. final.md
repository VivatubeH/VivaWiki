# final 키워드

## 핵심 개념
- 변수에 단 한 번만 값 할당 가능, 이후 변경 불가
- 참조형: 참조값(주소) 변경 불가, 객체 내부 값은 변경 가능
- 모든 인스턴스가 같은 값 → `static final` 상수로 선언

## 주요 사용법

### 지역 변수/매개변수
```java
final int data = 10;
// data = 20; // 컴파일 오류

void method(final int param) {
    // param = 20; // 컴파일 오류
}
```

### 필드 (생성자 초기화)
```java
public class ConstructInit {
    final int value;
    
    public ConstructInit(int value) {
        this.value = value; // 생성자에서 단 한 번 초기화
    }
}
```

### 상수 (static final)
```java
public class Constant {
    public static final int MAX_USERS = 1000;
    public static final double PI = 3.14;
}
```

### 참조형 변수
```java
final Data data = new Data();
// data = new Data(); // 참조값 변경 불가

data.value = 10; // 객체 내부 값은 변경 가능
data.value = 20; // OK
```

## 면접 대비 핵심

**Q1. final 키워드의 역할은?**
- 변수에 단 한 번만 값을 할당할 수 있게 제한. 이후 값 변경 시 컴파일 오류 발생. 불변성을 보장하여 안전한 코드 작성 가능.

**Q2. static final과 그냥 final의 차이는?**
- `final`: 인스턴스마다 다른 값 가능 (생성자 초기화 시)
- `static final`: 클래스 레벨 상수, 모든 인스턴스가 공유하는 단 하나의 값. 메모리 효율적.

**Q3. 참조형 변수에 final을 붙이면?**
- 참조값(주소) 자체는 변경 불가 → 다른 객체 참조 못함
- 하지만 참조 대상 객체의 내부 필드 값은 변경 가능

**Q4. 언제 final을 사용하는가?**
- 변경되면 안 되는 값 (회원 ID, 설정 상수 등)
- 매직 넘버 제거 및 중앙 관리
- 불변 객체 설계 시

**Q5. 상수 네이밍 관례는?**
- `static final` + 대문자 + 언더스코어 구분
- 예: `MAX_USERS`, `HOURS_IN_DAY`
