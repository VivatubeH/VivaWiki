# 기본형과 참조형

## 핵심 개념

**기본형(Primitive Type)**: 값 자체를 저장 (int, long, double, boolean)   
**참조형(Reference Type)**: 객체의 메모리 주소를 저장 (클래스, 배열)
```java
int a = 10;           // 기본형: 10이라는 값 자체 저장
Student s = new Student();  // 참조형: 객체 주소(x001) 저장
```

## 주요 차이점

| 구분 | 기본형 | 참조형 |
|------|--------|--------|
| 저장값 | 실제 값 | 참조값(주소) |
| 연산 | 가능 | 불가 |
| null | 불가 | 가능 |
| 시작 | 소문자 | 대문자 |

## 대입의 핵심 원칙

**자바는 항상 변수의 값을 복사한다** (기본형/참조형 동일)
```java
// 기본형: 값 10이 복사됨
int a = 10;
int b = a;  // b에 10 복사
a = 20;     // a만 변경 (b는 10 유지)

// 참조형: 주소 x001이 복사됨
Student s1 = new Student();  // x001
Student s2 = s1;             // s2에도 x001 복사
s1.name = "김철수";          // s2.name도 "김철수" (같은 객체)
```

## 메서드 호출 시 동작
```java
// 기본형 전달: 값만 복사 → 원본 변경 안됨
void change(int x) {
    x = 20;  // 매개변수만 변경
}
int a = 10;
change(a);  // a는 여전히 10

// 참조형 전달: 주소 복사 → 원본 변경됨
void change(Data data) {
    data.value = 20;  // 같은 객체 변경
}
Data d = new Data();
d.value = 10;
change(d);  // d.value가 20으로 변경!
```

## 변수 초기화
```java
// 멤버 변수: 자동 초기화 (0, false, null)
class Data {
    int value;      // 자동으로 0
    String name;    // 자동으로 null
}

// 지역 변수: 수동 초기화 필수
void method() {
    int a;          // 초기화 안하면
    System.out.println(a);  // 컴파일 에러!
}
```

## null과 NullPointerException
```java
Data data = null;           // 참조할 객체 없음
data.value = 10;            // NullPointerException!

// 해결: 객체 생성 후 할당
data = new Data();
data.value = 10;            // 정상 작동
```

**가비지 컬렉션**: 아무도 참조 안하는 객체는 JVM이 자동 제거

## 면접 대비 핵심

**Q1. 기본형과 참조형의 가장 큰 차이는?**
- 기본형은 값 자체를 저장하고, 참조형은 객체의 메모리 주소를 저장합니다
- 메서드 호출 시 기본형은 값이 복사되어 원본이 변경되지 않지만, 참조형은 주소가 복사되어 같은 객체를 가리키므로 원본이 변경될 수 있습니다

**Q2. 메서드에서 객체의 값을 변경하면 원본도 바뀌는 이유는?**
- 자바는 항상 값을 복사해서 전달합니다. 참조형의 경우 복사되는 값이 "객체의 주소"이기 때문에, 메서드의 매개변수와 원본 변수가 같은 객체를 가리키게 됩니다. 따라서 메서드 내부에서 객체를 수정하면 원본에도 반영됩니다

**Q3. NullPointerException은 언제 발생하나요?**
- null 값을 가진 참조 변수에 점(.)을 찍어서 접근하려 할 때 발생합니다
- 멤버 변수(참조형)는 자동으로 null로 초기화되므로, 객체를 생성해서 할당하지 않으면 NPE가 발생할 수 있습니다

**Q4. 가비지 컬렉션(GC)이란?**
- JVM이 더 이상 참조되지 않는 객체를 자동으로 메모리에서 제거하는 기능입니다
- 개발자가 메모리를 직접 관리하지 않아도 되어 메모리 누수를 방지합니다

**Q5. 지역 변수와 멤버 변수의 초기화 차이는?**
- 멤버 변수: 객체 생성 시 자동 초기화 (숫자 0, boolean false, 참조형 null)
- 지역 변수: 반드시 수동으로 초기화해야 하며, 초기화하지 않으면 컴파일 에러 발생
