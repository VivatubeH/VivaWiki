# 클래스와 객체

## 핵심 개념

**클래스**: 객체를 만들기 위한 설계도 (속성 + 기능 정의)  
**객체**: 클래스로부터 실제 메모리에 생성된 실체  
**인스턴스**: 특정 클래스로 생성된 객체 (객체와 같은 의미, 소속 강조)
```java
// 클래스 정의
public class Student {
    String name;  // 멤버 변수(필드)
    int age;
    int grade;
}

// 객체 생성
Student student1 = new Student();  // student1은 Student의 인스턴스
student1.name = "학생1";
```

---

## 주요 문법

### 1. 참조값과 메모리
```java
Student s1 = new Student();  // s1에는 참조값(주소) 저장
Student s2 = s1;             // 참조값 복사 → 같은 객체 가리킴
s2.name = "변경";            
System.out.println(s1.name); // "변경" - 같은 객체라서!
```

### 2. 배열 + 향상된 for문
```java
Student[] students = new Student[2];
students[0] = new Student();

for (Student s : students) {  // 배열 순회 간결 문법
    System.out.println(s.name);
}
```

### 3. null 처리
```java
Student s = null;  // 참조값 없음
s.name = "에러";   // NullPointerException 발생!
```

## 면접 대비 핵심

**Q1. 클래스와 객체의 차이는?**
- 클래스: 붕어빵 틀 (설계도)
- 객체: 실제 붕어빵 (메모리에 생성된 실체)
- 하나의 클래스로 여러 객체 생성 가능

**Q2. 참조형 변수 대입 시 주의사항은?**
- 자바는 항상 값을 복사하는데, 참조형은 참조값(주소)을 복사
- 결과: 두 변수가 같은 객체를 가리켜서 한쪽 수정이 양쪽 반영됨
- 실무에서 의도치 않은 공유 버그의 주요 원인

**Q3. NullPointerException이 발생하는 이유는?**
- null 상태 변수로 객체 접근 시도 (참조할 객체가 없음)
- 해결: 사용 전 null 체크 또는 Optional 사용 (Java 8+)

**Q4. 향상된 for문은 언제 사용?**
- 배열/컬렉션 전체 순회 시 (인덱스 불필요)
- 읽기 전용일 때 권장 (수정은 일반 for문)
