# 생성자와 this

## 핵심 개념

**생성자(Constructor)**
- 객체 생성 시점에 자동 호출되는 특별한 메서드
- 클래스명과 동일한 이름, 반환 타입 없음
- 객체 생성과 초기화를 한 번에 처리
```java
public class Member {
    String name;
    int age;
    
    // 생성자
    Member(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

// 사용
Member member = new Member("user1", 20);  // 생성과 초기화 동시에
```

**this 키워드**
- 인스턴스 자신의 참조값
- 매개변수와 멤버 변수 이름이 같을 때 구분 용도
```java
void init(String name) {
    this.name = name;  // this.name(멤버), name(매개변수)
}
```

## 주요 문법

### 1. 기본 생성자 규칙
```java
public class Member {
    String name;
    // 생성자 없음 → 자바가 기본 생성자 자동 생성
}

public class Member {
    String name;
    
    Member(String name) {  // 생성자 있음
        this.name = name;
    }
    // 기본 생성자 자동 생성 안됨 → 직접 만들어야 함
}
```

### 2. 생성자 오버로딩
```java
public class Member {
    String name;
    int age;
    int grade;
    
    Member(String name, int age) {
        this.name = name;
        this.age = age;
        this.grade = 50;  // 기본값
    }
    
    Member(String name, int age, int grade) {
        this.name = name;
        this.age = age;
        this.grade = grade;
    }
}
```

### 3. this()로 생성자 호출
```java
public class Member {
    String name;
    int age;
    int grade;
    
    Member(String name, int age) {
        this(name, age, 50);  // 다른 생성자 호출 (반드시 첫 줄)
    }
    
    Member(String name, int age, int grade) {
        this.name = name;
        this.age = age;
        this.grade = grade;
    }
}
```

## 면접 대비 핵심

**Q1. 생성자와 메서드의 차이는?**
- 생성자: 클래스명과 동일, 반환 타입 없음, 객체 생성 시 자동 호출
- 메서드: 임의의 이름, 반환 타입 필수, 명시적 호출 필요
- 생성자는 객체 초기화 전용, 메서드는 일반 기능 수행

**Q2. 기본 생성자가 자동 생성되는 조건은?**
- 클래스에 생성자가 하나도 없을 때만 자동 생성
- 생성자가 하나라도 있으면 기본 생성자 자동 생성 안됨
- 이유: 개발자가 명시적으로 생성자를 정의했다면 기본 생성자도 필요 시 직접 만들어야 한다는 설계 철학

**Q3. this와 this()의 차이는?**
- this: 인스턴스 자신의 참조값 (멤버 변수/메서드 접근)
- this(): 같은 클래스의 다른 생성자 호출 (반드시 첫 줄에 작성)
- 목적이 완전히 다른 별개의 기능

**Q4. 생성자 사용의 장점은?**
- 객체 생성과 초기화를 한 번에 처리 (편의성)
- 필수 값 입력 강제로 유효하지 않은 객체 생성 방지 (안정성)
- 컴파일 타임에 오류 확인 가능 (안전성)

**Q5. this()를 첫 줄에만 쓸 수 있는 이유는?**
- 객체가 완전히 초기화되기 전에 다른 코드가 실행되면 안 되기 때문
- 생성자 호출은 객체 초기화의 가장 먼저 일어나야 할 작업
- 일관된 초기화 순서 보장
