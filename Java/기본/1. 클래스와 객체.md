# 클래스와 객체

## 클래스가 필요한 이유

### 변수만 사용했을 때의 문제점
```java
String student1Name = "학생1";
int student1Age = 15;
int student1Grade = 90;

String student2Name = "학생2";
int student2Age = 16;
int student2Grade = 80;
```

- 학생이 추가될 때마다 변수를 3개씩 선언해야 함
- 한 학생의 데이터가 여러 변수에 흩어져 있어 관리가 어려움

---

## 클래스 도입

### 클래스 정의
```java
public class Student {
    String name;
    int age;
    int grade;
}
```

### 클래스 사용
```java
Student student1 = new Student();
student1.name = "학생1";
student1.age = 15;
student1.grade = 90;

Student student2 = new Student();
student2.name = "학생2";
student2.age = 16;
student2.grade = 80;
```

## 용어 정리

### 클래스 (Class)
- 객체를 생성하기 위한 설계도, 틀
- 속성(변수)과 기능(메서드)을 정의

### 객체 (Object)
- 클래스를 기반으로 실제 메모리에 생성된 실체
- 서로 독립적인 상태를 가짐

### 인스턴스 (Instance)
- 특정 클래스로부터 생성된 객체
- 객체와 같은 의미이지만, 어떤 클래스의 객체인지 강조할 때 사용
- 예: "student1은 Student의 인스턴스다"

### 멤버 변수 (Member Variable) / 필드 (Field)
- 클래스에 소속된 변수
- 예: `name`, `age`, `grade`

## 객체 생성과 참조

### 객체 생성 과정
```java
// 1. 변수 선언
Student student1;

// 2. 객체 생성
student1 = new Student();

// 한 줄로 작성
Student student1 = new Student();
```

### 참조값

- `new` 키워드로 객체를 생성하면 메모리 어딘가에 객체가 생성되고, 그 위치를 가리키는 **참조값(주소)**이 반환됨
- 변수에는 객체 자체가 아니라 **참조값**이 저장됨
```java
Student student1 = new Student(); // student1에는 참조값(예: x001)이 저장됨
```

### 참조값 확인
```java
System.out.println(student1);
// 출력 예: class1.Student@7a81197d
// @앞: 패키지.클래스, @뒤: 참조값(16진수)
```

## 객체 사용

### 멤버 변수 접근 - dot(.) 연산자
```java
// 값 대입
student1.name = "학생1";
student1.age = 15;
student1.grade = 90;

// 값 읽기
System.out.println("이름:" + student1.name);
System.out.println("나이:" + student1.age);
System.out.println("성적:" + student1.grade);
```

### 동작 원리
```java
student1.name = "학생1";
// 1. student1 변수에서 참조값(x001)을 읽음
// 2. 참조값을 통해 실제 객체에 접근
// 3. 객체의 name 필드에 "학생1" 저장
```

## 배열과 객체

### 객체 배열 생성
```java
// 배열 선언 및 생성
Student[] students = new Student[2];

// 객체 생성 후 배열에 저장
Student student1 = new Student();
student1.name = "학생1";
student1.age = 15;
student1.grade = 90;

Student student2 = new Student();
student2.name = "학생2";
student2.age = 16;
student2.grade = 80;

students[0] = student1;
students[1] = student2;
```

### 배열 선언 축약
```java
// 방법 1
Student[] students = new Student[]{student1, student2};

// 방법 2 (선언과 동시에 초기화할 때)
Student[] students = {student1, student2};
```

## 반복문 활용

### 일반 for문
```java
for (int i = 0; i < students.length; i++) {
    System.out.println("이름:" + students[i].name + 
                     " 나이:" + students[i].age + 
                     " 성적:" + students[i].grade);
}
```

### 향상된 for문 (Enhanced For Loop)
```java
for (Student s : students) {
    System.out.println("이름:" + s.name + 
                     " 나이:" + s.age + 
                     " 성적:" + s.grade);
}
```

## 주의사항

### 참조값 복사
```java
Student s1 = new Student();
s1.name = "학생1";

Student s2 = s1;  // 참조값만 복사됨
s2.name = "학생2";

System.out.println(s1.name);  // "학생2" - 같은 객체를 가리킴
```

- 자바에서 변수 대입은 항상 **값을 복사**
- 참조형 변수는 참조값을 가지므로, 대입 시 참조값이 복사됨
- 결과적으로 두 변수가 같은 객체를 가리키게 됨

### null
```java
Student student = null;  // 참조값이 없음
System.out.println(student.name);  // NullPointerException 발생
```

- `null`: 참조할 객체가 없다는 의미
- null 상태에서 객체에 접근하면 `NullPointerException` 발생
