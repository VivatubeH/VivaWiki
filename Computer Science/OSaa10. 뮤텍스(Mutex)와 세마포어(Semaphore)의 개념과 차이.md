뮤텍스와 세마포어가 필요한 이유
-----------------------------------
- 운영체제에서 여러 개의 프로세스나 스레드가 동시에 자원을 사용할 때, 경쟁 상태(Race Condition)가 발생할 수 있습니다.
- 예를 들면, 두 사람이 동시에 통장에서 출금하게 되면 잔고가 꼬일 수 있습니다.
- 그래서 공유 자원에 대해 한 번에 하나만 접근할 수 있도록 하는 동기화(Synchronization)이 필요합니다.

뮤텍스(Mutex)
----------------------
- Mutual Exclusion(상호 배제)의 줄임말입니다.
- 하나의 자원에 하나의 스레드만 접근가능하게 하는 것을 뮤텍스라고 합니다.
- 락(lock)과 언락(unlock)의 구조로 작동합니다. ( 자원을 쓰려면 lock을 획득해야 하고, 다 쓰면 unlock으로 해제해야 합니다. )

```
lock(mutex)
// 자원 접근
unlock(mutex)
```

세마포어(Semaphore)
---------------------------------
- 뮤텍스를 확장한 개념으로 정해진 수만큼 동시에 접근을 허용하는 것이 세마포어(Semaphore)입니다.
- 세마포어는 정수 값(counter)를 가지며, 0이면 대기 0보다 크면 통과가 가능합니다.

비유로 보는 뮤텍스와 세마포어
---------------------------------
- 뮤텍스는 1칸의 화장실과 같습니다. 한 명만 사용가능하고, 문을 잠그고 들어가서 들어간 사람만이 직접 잠금 해제를 할 수 있습니다.
- 세마포어는 자리가 N개인 주차장과 같습니다. N명까지 동시에 사용 가능하고, 자리가 없다면 밖에서 기다립니다.
