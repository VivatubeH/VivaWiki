데이터베이스(Databae)
---------------------------------
데이터베이스는 많은 데이터를 효율적으로 관리하고 활용할 수 있는 저장소를 일컫는 용어입니다.

데이터베이스의 필요성 
--------------------------------------------
- 데이터 무결성 보장: 데이터를 정확하고 일관되게 관리해서, 잘못된 데이터 입력을 방지합니다.
- 데이터 중복 최소화 : 중복 데이터를 최소화해서 저장 공간을 절약하고, 데이터 관리 비용을 줄입니다.
- 데이터 공유성 제공 : 여러 사용자나 여러 애플리케이션이 데이터를 공유해서 사용할 수 있게 해줍니다.
- 데이터 보안성 강화 : 데이터 접근 권한을 관리해서 중요 정보 유출을 방지할 수 있게 해줍니다.

데이터베이스 관리 시스템(DBMS)
--------------------------------------------
- DBMS(Database Management System)은 데이터베이스를 관리하고 운영하는 소프트웨어입니다.
- 데이터를 효율적으로 저장, 수정, 삭제, 검색할 수 있도록 도와줍니다.

DBMS의 주요 역할
--------------------------------
- 데이터 정의(DDL) : 데이터 구조를 정의합니다. ( 테이블, 인덱스 등... )
- 데이터 조작(DML) : 데이터를 삽입, 수정, 삭제, 조회합니다.
- 데이터 제어(DCL) : 데이터 접근 권한을 관리합니다.

데이터베이스 모델 - 관계형 데이터베이스 (RDBMS)
--------------------------------------------------------------
- RDBMS는 데이터를 테이블 형태로 관리하는 모델입니다. ( 각 테이블은 행(row)와 열(column)으로 구성됩니다. )
- 키(Key)를 통해서 데이터의 무결성을 보장합니다.
- 대표적인 DBMS로는 Oracle, MySQL, PostgreSQL 등이 있습니다.

데이터베이스 모델 - 비관계형 데이터베이스(NoSQL)
----------------------------------------------------------------
- 유연한 데이터 구조를 지원해서 다양한 형태의 데이터를 관리합니다.
- 문서(Document), 키-값(Key-Value), 그래프(Graph) 등 다양한 모델을 가집니다.
- 대표적인 DBMS로는 MongoDB, Redis가 있습니다.

SQL(Structured Query Language)
------------------------------------------------
- SQL은 데이터베이스에서 데이터를 조작하고 관리하는 언어를 말합니다.

SQL의 종류
----------------------------------------------
- DDL(Data Definition Language) : CREATE, ALTER, DROP / 데이터 구조를 정의합니다.
- DML(Data Manipulation Language) : SELECT, INSERT, UPDATE, DELETE / 데이터를 조작합니다.
- DCL(Data Control Language) : GRANT, REVOKE / 데이터 접근을 제어합니다.
- TCL(Transaction Control Language) : COMMIT, ROLLBACK, SAVEPOINT / 트랜잭션을 제어합니다.

데이터 무결성
-----------------------------------------------
- 데이터의 정확성, 일관성, 유효성을 보장하는 것을 데이터 무결성이라고 합니다.
- 참조 무결성 : 외래 키 제약 조건을 통해 부모-자식 관계를 보장합니다.
- 도메인 무결성 : 데이터 타입이나 형식을 강제합니다.

트랜잭션 관리
----------------------------------------------
- 데이터베이스의 작업 단위를 트랜잭션이라고 합니다.

트랜잭션의 기본 특성(ACID 속성)
-----------------------------------------
- Atomicity(원자성) : 모두 수행되거나 모두 수행되지 않아야 합니다.
- Consistency(일관성) : 일관성 있는 데이터 상태를 보장해야 합니다.
- Isolation(고립성) : 트랜잭션 간 간섭을 방지해야 합니다.
- Durability(지속성) : 트랜잭션 완료 후 데이터를 영구 저장해야 합니다.
